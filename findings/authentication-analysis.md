# Authentication Investigation Findings

This document summarizes the results of the authentication activity investigation performed using Splunk SIEM.

The objective was to identify suspicious login behavior generated by simulated attacks within the Windows system.

---

# Investigation Overview

Authentication logs from the Windows Security Event Log were ingested into Splunk and analyzed using SPL queries.

Two attack simulations were conducted:

1. Brute-force login attempts
2. Password spray attack simulation

These activities generated authentication events that were analyzed to detect abnormal login behavior.

---

# Key Observations

### Failed Login Attempts

Multiple authentication failures were observed during the brute-force simulation.

These failures generated:

Event ID **4625**

This event indicates that a user attempted to log in but authentication failed.

---

### Successful Logins

Successful login events were also present within the dataset.

These generated:

Event ID **4624**

Successful authentication events help analysts confirm normal login activity.

---

### Password Spray Indicators

During the password spray simulation, authentication failures occurred across **multiple accounts** within a short time frame.

This pattern differs from brute-force attacks and is a common technique used by attackers attempting to bypass account lockout protections.

The attack generated repeated **4625 authentication failures** across different usernames.

---

# Security Impact

Authentication attacks such as brute-force and password spraying can lead to:

- Unauthorized system access
- Credential compromise
- Privilege escalation
- Lateral movement inside a network

Detecting abnormal authentication patterns is therefore critical for Security Operations Centers.

---

# Conclusion

The Splunk SIEM successfully detected authentication activity generated by the simulated attacks.

Using log analysis and statistical queries, suspicious login behavior was identified and analyzed.

This demonstrates how SIEM platforms can be used to monitor authentication activity and detect potential account compromise attempts.
